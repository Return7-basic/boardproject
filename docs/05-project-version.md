## 버전 관리 규칙

### Semantic Versioning (SemVer)

프로젝트는 [Semantic Versioning](https://semver.org/) 규칙을 따릅니다.

**형식**: `v{MAJOR}.{MINOR}.{PATCH}`

- **MAJOR**: 호환되지 않는 API 변경
- **MINOR**: 하위 호환성을 유지하면서 기능 추가
- **PATCH**: 하위 호환성을 유지하면서 버그 수정

### 예시

- `v1.0.0`: 첫 번째 정식 릴리스
- `v1.1.0`: 새로운 기능 추가 (댓글 채택 기능)
- `v1.1.1`: 버그 수정
- `v2.0.0`: 주요 변경사항 (API 변경)
